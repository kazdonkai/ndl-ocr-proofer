```bash
cd /Users/kazumikaizuka/Obsidian/Minji/proofreading-app
./start.sh
```

ブラウザで `http://localhost:5173` を開く。

---

## 基本的な校正フロー

1. 左サイドバーから文書ファイルを選択
2. テキストエリアで OCR テキストを確認・修正
3. AI 解析ボタンで校正候補を表示
4. 候補を確認して承認 or 却下

---

## 手動訂正と temporary 登録

1. テキスト上でテキストを選択 → 手動訂正ダイアログが開く
2. 正しい表記を入力して確定
3. 元テキストと異なる場合、**「temporary に登録」プロンプトが続いて表示される**
4. term・normalized・category（任意）・note（任意）を確認して「登録」ボタン
   - 同じ term を後で再登録すると `approval_count` が上がる

## 辞書管理モーダルの使い方

辞書管理ボタン（ヘッダー）から開く。

### temporary タブ
| 操作 | 説明 |
|------|------|
| エントリ一覧 | 全 temporary ファイルの内容が表示される |
| 有効/無効切り替え | enabled フラグで一時的に辞書から除外できる |
| 編集 | term・normalized 等のフィールドを変更できる |
| 削除 | エントリを物理削除する |
| ▲ 昇格 | approval 辞書へ昇格する（approval ファイルを選択） |
| 昇格候補バッジ | `approval_count >= 3` のエントリに表示（リスト先頭にソート）。**自動昇格ではない** |

### approval タブ
| 操作 | 説明 |
|------|------|
| エントリ追加 | approval 辞書に直接追加できる |
| 編集・削除 | 既存エントリを操作できる |

### ファイル管理
- 「新規辞書ファイル作成」で temporary / approval どちらでも空ファイルを作れる
- ファイル削除は protect=true エントリがある場合は拒否される
- `user_manual_temporary.csv` は削除不可（基幹ファイル）

---

## 昇格候補について

> **昇格候補は approval への自動昇格ではありません。**

同一 term が 3 回以上再登録されると **昇格候補バッジ** が表示されるだけです。  
approval への移行は **▲ 昇格ボタンを手動で押す** ことで行います。

昇格するとその term は temporary から削除され、approval ファイルに `source=promoted` として追加されます。

## バックアップ

- 辞書ファイルへの書き込みが発生するたびに、自動的に `archive/old_versions/` へバックアップが作成されます
- 手動バックアップも辞書管理モーダルから実行できます
